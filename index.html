模块 02

热门经典故事库（Story Vault）

将热门经典电视剧 & 电影中的高传播记忆点转写为文字分镜，整理成可直接喂给 AI 的故事素材，确保生成效果稳定。

选取原则

情绪峰值 + 传播度 + 版权安全

优先提取 30 秒-90 秒的情绪高点或反转桥段，标注情绪线、冲突点与出圈台词，只保留结构化描述避免版权侵权。

喂给 AI 的格式

文字分镜 · 角色变量 · 风格提示

每个故事包含 4-8 个镜头节点，提供镜头景别、角色行为、台词/旁白、BGM/气氛提示，并预留 {{role}}、{{location}} 变量。

应用入口

一键选用 · 复制模板

点击“加入生成”即可把分镜结构写入生成队列；“复制分镜模板”复制标准 JSON/Prompt 片段，方便直接调用模型。

类型：爱情 / 家庭 / 励志 / 悬疑 来源：电视剧 / 电影 时长：30s / 60s / 90s 变量：角色名、地点、时代背景 质量：共鸣度 ≥ 8 /10 · 模型可控

数据表

video_jobs / ai_jobs

字段：id, user_id, provider, provider_job_id, status, input_script, style_params (JSON), output_video_url, error_message, created_at, updated_at。

  • 创建任务时写入 status=pending，调用 Provider 后更新为 running。
  • 轮询返回 succeeded 或 failed 时，写入 output_video_url / error_message。
  • 默认 Provider 为 Mock，可扩展 Runway/Pika/Sora 适配器。

轮询 Worker

后台状态同步

每 10 秒扫描 pending / running 任务，通过 Provider.getJobStatus() 更新状态。

  • 成功：写入视频 URL，标记 succeeded。
  • 失败：写入错误信息，标记 failed。
  • 静态站点也可使用该 Worker 模拟外部 API 行为。
